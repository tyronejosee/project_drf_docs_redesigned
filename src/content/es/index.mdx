# Django REST Framework

Django REST framework es un conjunto de herramientas potente y flexible para construir APIs web.

Algunas razones por las que podrías querer usar REST framework:

- La API navegable es una gran ventaja de usabilidad para tus desarrolladores.
- [Políticas de autenticación][authentication] que incluyen paquetes para [OAuth1a][oauth1-section] y [OAuth2][oauth2-section].
- [Serialización][serializers] que admite tanto fuentes de datos [ORM][modelserializer-section] como [no ORM][serializer-section].
- Personalizable en todos los niveles - simplemente usa [vistas basadas en funciones regulares][functionview-section] si no necesitas las características [más][generic-views] [potentes][viewsets] de los [enrutadores][routers].
- Documentación extensa y [gran soporte de la comunidad][group].
- Utilizado y confiado por empresas reconocidas internacionalmente como [Mozilla][mozilla], [Red Hat][redhat], [Heroku][heroku] y [Eventbrite][eventbrite].

## Requisitos

REST framework requiere lo siguiente:

- Django (4.2, 5.0, 5.1, 5.2)
- Python (3.9, 3.10, 3.11, 3.12, 3.13)

**Recomendamos encarecidamente** y solo admitimos oficialmente la última versión de parche de cada serie de Python y Django.

Los siguientes paquetes son opcionales:

- [PyYAML][pyyaml], [uritemplate][uriteemplate] (5.1+, 3.0.0+) - Soporte para generación de esquemas.
- [Markdown][markdown] (3.3.0+) - Soporte para Markdown en la API navegable.
- [Pygments][pygments] (2.7.0+) - Añade resaltado de sintaxis al procesamiento de Markdown.
- [django-filter][django-filter] (1.0.1+) - Soporte para filtrado.
- [django-guardian][django-guardian] (1.1.1+) - Soporte para permisos a nivel de objeto.

## Instalación

Instala usando `pip`, incluyendo los paquetes opcionales que necesites...

```bash
pip install djangorestframework
pip install markdown       # Soporte para Markdown en la API navegable
pip install django-filter  # Soporte para filtrado
```

...o clona el proyecto desde GitHub.

```bash
git clone https://github.com/encode/django-rest-framework
```

Añade `'rest_framework'` a tu configuración `INSTALLED_APPS`.

```python
INSTALLED_APPS = [
    ...
    'rest_framework',
]
```

Si planeas usar la API navegable, probablemente también quieras añadir las vistas de inicio y cierre de sesión de REST framework. Añade lo siguiente a tu archivo raíz `urls.py`:

```python
urlpatterns = [
    ...
    path('api-auth/', include('rest_framework.urls'))
]
```

Ten en cuenta que la ruta de la URL puede ser la que prefieras.

## Ejemplo

Veamos un ejemplo rápido de cómo usar REST framework para construir una API simple basada en modelos.

Crearemos una API de lectura/escritura para acceder a la información de los usuarios de nuestro proyecto.

Todas las configuraciones globales para una API de REST framework se guardan en un único diccionario de configuración llamado `REST_FRAMEWORK`. Comienza añadiendo lo siguiente a tu módulo `settings.py`:

```python
REST_FRAMEWORK = {
    # Usa los permisos estándar de `django.contrib.auth`,
    # o permite acceso de solo lectura a usuarios no autenticados.
    'DEFAULT_PERMISSION_CLASSES': [
        'rest_framework.permissions.DjangoModelPermissionsOrAnonReadOnly'
    ]
}
```

No olvides asegurarte de que has añadido `rest_framework` a tu `INSTALLED_APPS`.

Ahora estamos listos para crear nuestra API. Aquí está el módulo raíz `urls.py` de nuestro proyecto:

```python
from django.urls import path, include
from django.contrib.auth.models import User
from rest_framework import routers, serializers, viewsets

# Los serializadores definen la representación de la API.
class UserSerializer(serializers.HyperlinkedModelSerializer):
    class Meta:
        model = User
        fields = ['url', 'username', 'email', 'is_staff']

# Los ViewSets definen el comportamiento de la vista.
class UserViewSet(viewsets.ModelViewSet):
    queryset = User.objects.all()
    serializer_class = UserSerializer

# Los enrutadores proporcionan una forma fácil de determinar automáticamente la configuración de URL.
router = routers.DefaultRouter()
router.register(r'users', UserViewSet)

# Conecta nuestra API usando el enrutamiento automático de URL.
# Además, incluimos las URLs de inicio de sesión para la API navegable.
urlpatterns = [
    path('', include(router.urls)),
    path('api-auth/', include('rest_framework.urls', namespace='rest_framework'))
]
```

Ahora puedes abrir la API en tu navegador en [http://127.0.0.1:8000/](http://127.0.0.1:8000/) y ver tu nueva API de 'usuarios'. Si usas el control de inicio de sesión en la esquina superior derecha, también podrás agregar, crear y eliminar usuarios del sistema.
